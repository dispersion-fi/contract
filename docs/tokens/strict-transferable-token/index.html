    tokens/StrictTransferableToken.vy #

    vyper: 0.2.11 author: Dispersion Finance Team license: MIT

    Strict Transferable Token

    Based on the ERC-20 token that cannot be freely transferred. Transfer awailable for contracts and accounts from transferableAccountCheckList only. decimals() = 18.

    Burnable and Mintable ERC20 token.

    Events #

    Transfer

    • sender : address, indexed
    • receiver : address, indexed
    • value : uint256, notIndexed

    Approval

    • owner : address, indexed
    • spender : address, indexed
    • value : uint256, notIndexed

    CommitOwnership

    • owner : address, notIndexed

    ApplyOwnership

    • owner : address, notIndexed

    Methods #

    init #

    type: nonpayable constructor

    Contract constructor.

    owner = msg.sender

    Arguments:

    • _name: - Token full name

    • _symbol: - Token symbol

    • _mintersCheckList: - Minters check list. List of addresses that are allowed to mint coins

    • _transferableAccountCheckList: - Transferable account check list, who is allowed to transfer tokens

    decimals #

    type: view function gas: 288

    Returns token decimals value.

    For ERC20 compatibility.

    Returns:

    • _0 - Token decimals

    setName #

    type: nonpayable function gas: 142531

    Changes token name and symbol to _name and _symbol.

    Callable by owner only.

    Arguments:

    • _name: - New token name

    • _symbol: - New token symbol

    mint #

    type: nonpayable function gas: 75984

    Mints new tokens for account _account with amount _amount.

    Callable by minter only. Emits a Transfer event originating from ZERO_ADDRESS. _account can't be equal ZERO_ADDRESS.

    Arguments:

    • _account: - Account to mint tokens for

    • _amount: - Amount to mint

    transfer #

    type: nonpayable function gas: 76594

    Transfers _amount of tokens from msg.sender to _recipient address

    ERC20 function. Emits a Transfer event with msg.sender, _recipient, _amount. _recipient can't be equal ZERO_ADDRESS

    Arguments:

    • _recipient: - Account to send tokens to

    • _amount: - Amount to send

    Returns:

    • _0 - Boolean success value

    transferFrom #

    type: nonpayable function gas: 76415

    Transfers _amount of tokens from _sender to _recipient address

    ERC20 function. Allowance from _sender to msg.sender is needed. Emits a Transfer event with _sender, _recipient, _amount. _sender and _recipient can't be equal ZERO_ADDRESS

    Arguments:

    • _sender: - Account to send tokens from

    • _recipient: - Account to send tokens to

    • _amount: - Amount to send

    Returns:

    • _0 - Boolean success value

    approve #

    type: nonpayable function gas: 538

    Approves allowance from msg.sender to _spender address for _amount of tokens

    For ERC20 compatibility.

    Arguments:

    • _spender: - Allowed account to send tokens from msg.sender

    • _amount: - Allowed amount to send tokens from msg.sender

    Returns:

    • _0 - False always

    allowance #

    type: view function gas: 668

    Returns allowance value.

    For ERC20 compatibility.

    Returns:

    • _0 - 0 always

    burn #

    type: nonpayable function gas: 74651

    Burn _value tokens belonging to msg.sender

    Emits a Transfer event with a destination of ZERO_ADDRESS

    Arguments:

    • _amount: - Amount to burn

    transferOwnership #

    type: nonpayable function gas: 37861

    Transfers ownership by setting new owner _futureOwner candidate

    Callable by owner only. Emit CommitOwnership event with _futureOwner

    Arguments:

    • _futureOwner: - Future owner address

    applyOwnership #

    type: nonpayable function gas: 38717

    Applies transfer ownership

    Callable by owner only. Function call actually changes owner. Emits ApplyOwnership event with _owner
